## 2. Lists (`list`)
A **list** is a mutable, ordered collection of heterogeneous elements.

### Characteristics:
- **Mutable**: Elements can be modified, added, or removed in-place.
- **Ordered & Indexed**: Maintains insertion order and supports indexing/slicing.
- **Allows Duplicates**: Multiple occurrences of the same element are allowed.
- **Dynamic & Heterogeneous**: Can grow or shrink, and hold elements of different data types simultaneously.

### Common List Methods:

| Method | Description | Example |
| :--- | :--- | :--- |
| `.append(x)` | Appends element `x` to the end of the list | `lst.append(5)` |
| `.extend(iterable)` | Extends list by appending all items from the iterable | `lst.extend([2, 3])` |
| `.insert(index, x)` | Inserts element `x` at the specified index | `lst.insert(0, 9)` |
| `.remove(x)` | Removes the first occurrence of element `x` (raises `ValueError` if not found) | `lst.remove(5)` |
| `.pop(index)` | Removes and returns the element at the index (defaults to the last element) | `item = lst.pop()` |
| `.clear()` | Removes all items from the list | `lst.clear()` |
| `.index(x)` | Returns the zero-based index of the first occurrence of `x` | `idx = lst.index(5)` |
| `.count(x)` | Returns the number of occurrences of `x` | `num = lst.count(5)` |
| `.sort(key, reverse)` | Sorts the list in-place (ascending by default) | `lst.sort()` |
| `.reverse()` | Reverses the list elements in-place | `lst.reverse()` |
| `.copy()` | Returns a shallow copy of the list | `new_lst = lst.copy()` |

### Built-in Functions with Lists:
- `len(lst)`: Returns the number of items.
- `list(iterable)`: Converts an iterable to a list.
- `max(lst)` / `min(lst)`: Returns the largest/smallest element.
- `sum(lst)`: Sums up numeric elements.
- `sorted(lst)`: Returns a new sorted list, leaving original intact.
- `reversed(lst)`: Returns a reverse iterator over the list.
- `enumerate(lst)`: Returns an enumerate object yielding (index, value) tuples.
- `any(lst)` / `all(lst)`: Returns `True` if any/all elements evaluate to `True`.
## 3. Tuples (`tuple`)
A **tuple** is an immutable, ordered sequence of elements, typically used to store collections of heterogeneous data.

### Characteristics:
- **Immutable**: Elements cannot be modified, added, or removed after creation.
- **Ordered & Indexed**: Supports indexing and slicing.
- **Allows Duplicates**: Can contain multiple duplicate elements.
- **Performance & Safety**: Faster than lists and can be used as dictionary keys (since they are hashable).

### Common Tuple Methods:
Since tuples are immutable, they only support two methods:

| Method | Description | Example |
| :--- | :--- | :--- |
| `.count(x)` | Returns the number of occurrences of value `x` | `tup.count(5)` |
| `.index(x)` | Returns the index of the first occurrence of value `x` (raises `ValueError` if not found) | `tup.index(5)` |

### Built-in Functions with Tuples:
- `len(tup)`: Returns number of elements.
- `tuple(iterable)`: Converts an iterable to a tuple.
- `max(tup)` / `min(tup)` / `sum(tup)`: Standard numerical operations.
- `sorted(tup)`: Returns a new *sorted list* containing the tuple's elements.
## 4. Dictionaries (`dict`)
A **dictionary** is a mutable collection of key-value pairs where keys must be unique and immutable.

### Characteristics:
- **Mutable**: Key-value pairs can be added, updated, or removed.
- **Insertion-Ordered**: Since Python 3.7, dictionaries maintain the order in which keys were inserted.
- **Indexed by Key**: Values are accessed via their unique keys, not index positions.
- **Unique, immutable Keys**: Keys must be of immutable types (strings, numbers, tuples) and cannot be duplicated. Values can be anything.

### Common Dictionary Methods:

| Method | Description | Example |
| :--- | :--- | :--- |
| `.get(key, default)` | Safely retrieves value for `key`; returns `default` if key doesn't exist | `dict.get('age', 0)` |
| `.keys()` | Returns a dynamic view object of all dictionary keys | `dict.keys()` |
| `.values()` | Returns a dynamic view object of all dictionary values | `dict.values()` |
| `.items()` | Returns a dynamic view object of all `(key, value)` tuples | `dict.items()` |
| `.update(other)` | Updates the dictionary with key-value pairs from `other` (dict or iterable) | `dict.update({'gpa': 3.9})` |
| `.pop(key, default)` | Removes the key and returns its value; returns `default` if not found | `dict.pop('age')` |
| `.popitem()` | Removes and returns the last inserted `(key, value)` pair as a tuple | `k, v = dict.popitem()` |
| `.setdefault(key, default)` | If key exists, returns its value. Else, inserts key with value `default` and returns it | `dict.setdefault('status', 'active')` |
| `.clear()` | Removes all key-value pairs | `dict.clear()` |
| `.copy()` | Returns a shallow copy of the dictionary | `new_dict = dict.copy()` |

### Built-in Functions with Dictionaries:
- `len(d)`: Returns the number of key-value pairs.
- `dict(iterable)`: Creates a dictionary from an iterable of key-value pairs.
- `sorted(d)`: Returns a sorted list of the dictionary's *keys*.
