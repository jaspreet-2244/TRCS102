Before we can analyze data or build dashboards, we must first check if our data is **clean and reliable**. This process is called **Data Quality Assessment**.

> _"Garbage In, Garbage Out"_ — If the data is bad, every analysis built on it will be wrong.

---
## 📖 Concept Explanation

### What is Data Quality?

**Data Quality** refers to how accurate, complete, and reliable your data is. Before doing any analysis, we need to answer:

| Question | Why It Matters |
|----------|----------------|
| Are there **missing values**? | Missing data can give wrong averages, wrong counts, and misleading charts |
| Are there **duplicate rows**? | Duplicates inflate numbers — e.g., counting the same sale twice |
| Are the **data types correct**? | A date stored as text cannot be used for time-based analysis |
| Are **numerical values sensible**? | Negative prices or quantities of 0 indicate errors |
| Are **categorical values consistent**? | "Electronics" vs "electronics" vs "ELECTRONICS" are treated as different |
| Is the **data consistent**? | If Revenue should equal Price × Quantity, we must verify it |

### The Data Quality Assessment Workflow

```
Step 1: Load the Data
    ↓
Step 2: Get a Quick Overview (shape, info, head)
    ↓
Step 3: Check for Missing Values
    ↓
Step 4: Check for Duplicate Records
    ↓
Step 5: Inspect Numerical Columns
    ↓
Step 6: Inspect Categorical Columns
    ↓
Step 7: Inspect Date Columns
    ↓
Step 8: Verify Data Consistency
    ↓
Step 9: Write a Data Quality Report
```

### 🔍 1. Check Missing Values

**What are Missing Values?**

Missing values are blank or empty cells in the data. In Python, they appear as `NaN` (Not a Number) or `None`.

**Why are they a problem?**
- They can give wrong averages (e.g., average salary ignoring some employees)
- Many functions crash or give errors with missing data
- Charts and visualizations become misleading
```python
# ---- CHECK MISSING VALUES ----

# .isnull() marks each cell as True (missing) or False (not missing)
# .sum() counts the number of True values per column

missing_values = df.isnull().sum().sum()
print(missing_values)
```
```
16
```
```python
# Let's also calculate the PERCENTAGE of missing values
# This is more useful when the dataset has thousands of rows
# Formula: (missing count / total rows) * 100

print("\nMISSING VALUES — COUNT & PERCENTAGE")
missing_count = df.isnull().sum()             # Count of missing per column
missing_percent = (missing_count / len(df)) * 100  # Percentage of missing

# Create a summary table
missing_summary = pd.DataFrame({
    'Missing Count': missing_count,
    'Missing %': missing_percent.round(2)  # Round to 2 decimal places
})

print(missing_summary)
```

### 🔍 2. Check Duplicate Records

**What are Duplicates?**

A duplicate row is an **exact copy** of another row. For example, the same transaction recorded twice.

**Why are they a problem?**
- Revenue gets counted twice → wrong total sales
- Product counts get inflated
- Reports become inaccurate
```python
# ---- CHECK DUPLICATE RECORDS ----

# .duplicated() marks each row as True (duplicate) or False (not duplicate)
# .sum() counts the total number of duplicate rows

duplicate_count = df.duplicated().sum()
print(f"Number of duplicate rows: {duplicate_count}")
```

### 🔍 3. Inspect Numerical Columns

**What are Numerical Columns?**

Columns that contain numbers — like `Price`, `Quantity`, and `Revenue`.

**What to look for:**
- Are there **negative values** where they shouldn't be? (e.g., negative Price)
- Are there **zero values** that don't make sense? (e.g., Quantity = 0)
- Are there **extreme outliers**? (e.g., a laptop costing ₹1,00,000 when others cost ₹1,000)
- Do **min and max values** make sense?

Columns that contain **text or labels** — like `Product`, `Category`, and `Transaction_ID`.

**What to look for:**
- How many **unique values** are there?
- Are there **inconsistent names**? (e.g., "laptop" vs "Laptop" vs "LAPTOP")
- Which value appears **most/least often**?
- Are there any **unexpected values**?
