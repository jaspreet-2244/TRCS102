## Loops (Iteration)
Loops allow you to run a block of code repeatedly. Python provides two main types of loops:
1. **`for` loops**: Best when you know in advance how many times you want to iterate (definite iteration).
2. **`while` loops**: Best when you want to iterate until a specific condition changes (indefinite iteration).
### 2.1 The `for` Loop
A `for` loop is used to iterate over a sequence (such as a list, tuple, string, or range). It executes the loop body once for each item in the sequence.

#### Syntax:
```python
for item in sequence:
    # Code to run for each item
```
#### The `range()` Function:
In Python, the built-in `range()` function generates a sequence of numbers, which is extremely useful for running a loop a specific number of times.

Syntax: `range(start, stop, step)`
* `start` (optional, default `0`): The beginning number (inclusive).
* `stop` (required): The ending number (exclusive).
* `step` (optional, default `1`): The increment value.

#### Iterating Over a String:
Since a string is a sequence of characters, you can iterate over it letter-by-letter.
```python
word = "Data"
print(f"Letters in '{word}':")
for letter in word:
    print(letter,end=" ")
```
### 2.2 The `while` Loop
A `while` loop runs a block of code as long as a specified condition remains `True`.

#### Syntax:
```python
while condition:
    # Code to run
```

####  Important:
Inside a `while` loop, you must write code that will eventually make the condition `False`. Otherwise, you will create an **infinite loop**, causing your program to run forever!

#### How it works:
```text
           [Start]
              │
              ▼
   ┌──► Is condition True? ── No ──┐
   │          │                    │
   │         Yes                   │
   │          │                    │
   │          ▼                    │
   │    [Run loop-block]           │
   │          │                    │
   └──────────┘                    │
                                   ▼
                              [Exit Loop]
```
---
##  Loop Control Statements
Sometimes, you need to alter the standard execution of a loop. Python provides three keywords to control loops:
1. **`break`**: Exits the loop prematurely.
2. **`continue`**: Skips the rest of the current iteration and jumps directly to the next iteration.
3. **`pass`**: A null statement used as a placeholder.
### 3.1 The `break` Statement
The `break` statement immediately terminates the loop. Any code after `break` inside the loop is skipped, and the program moves to the statement right after the loop.
### 3.2 The `continue` Statement
The `continue` statement skips the rest of the code in the **current** iteration and moves control back to the top of the loop for the next iteration. It does not stop the entire loop, only the current cycle.
### 3.3 The `pass` Statement
The `pass` statement does absolutely nothing! It is a null operation. It is used as a syntax placeholder when a statement is required, but you haven't written the code for it yet.

#### Why do we need it?
Python will raise an error if an empty block is left under an `if` statement or a loop. Using `pass` prevents this syntax error.

##  Function Creation
A function is a block of organized, reusable code that is used to perform a single, related action. Functions provide better modularity for your application and a high degree of code reusability.

### 1.1 Defining a Function
In Python, we define a function using the `def` keyword, followed by the function name, parentheses `()`, and a colon `:`. The block of code inside the function must be indented.

#### Syntax:
```python
def function_name(parameters):
    """Docstring: Optional documentation describing the function."""
    # Function body
    statements
```

#### Key Terms:
* **Parameters**: The placeholders defined in the function declaration.
* **Arguments**: The actual values passed to the function when calling it.
* **Docstring**: Documentation enclosed in triple quotes `"""` that describes what the function does. It is highly recommended!
### 1.2 Function Execution Flow
When you define a function, Python reads it to check for syntax but does not execute the code inside. The code is only executed when the function is explicitly **called**.
---
##  Function Arguments
Arguments allow us to pass data into a function. Python is highly flexible with how arguments can be defined and passed.

We will explore four types of arguments:
1. **Positional Arguments**
2. **Keyword Arguments**
3. **Default Arguments**
4. **Variable-Length Arguments (`*args` and `**kwargs`)**
### 2.1 Positional & Keyword Arguments
* **Positional Arguments**: Must be passed in the correct order in which they were defined.
* **Keyword Arguments**: Passed by matching the parameter name. Order does not matter.
### 2.2 Default Arguments
You can assign default values to parameters. If an argument is not provided during the function call, the default value is used.

> ** Warning:** Default parameters must always follow non-default parameters in the function definition!
> `def greet(name, message="Hello")` is **valid**.
> `def greet(message="Hello", name)` is **invalid** and raises a syntax error.
### 2.3 Variable-Length Arguments (`*args` and `**kwargs`)
Sometimes you don't know in advance how many arguments will be passed to a function. Python handles this with:
* `*args`: Collects extra positional arguments as a **Tuple**.
* `**kwargs`: Collects extra keyword arguments as a **Dictionary**.
---
##  Return Statements
A `return` statement exits a function and sends a value (or multiple values) back to the caller.

### 3.1 Returning a Single Value vs None
If a function doesn't have a `return` statement (or has a bare `return`), it automatically returns `None`.

### 3.2 Returning Multiple Values
A function can return multiple values separated by commas. Python packs them and returns them as a **Tuple**. You can unpack them into separate variables.
---
##  Lambda Functions
A **Lambda function** is a small, anonymous function (a function without a name). It can take any number of arguments, but can only have **one expression**. The expression is evaluated and automatically returned.

#### Syntax:
```python
lambda arguments: expression
```

#### Why use Lambda functions?
They are useful for short-lived, throwaway functions, especially when passed as arguments to higher-order functions like `map()`, `filter()`, and `sorted()`.
---
##  Variable Scope
Variable scope refers to where in the program a variable is recognized and can be accessed.

In Python, scope follows the **LEGB Rule**:
1. **Local (L)**: Defined inside the function/class.
2. **Enclosing (E)**: Defined in outer nested functions.
3. **Global (G)**: Defined at the top level of the module.
4. **Built-in (B)**: Names pre-defined in Python (e.g. `print`, `sum`).

### 5.1 Local vs Global Demonstration
```python
global_var = "I am global"

def scope_demo():
    local_var = "I am local"
    print("Inside function:")
    print("- Accessing global variable:", global_var)
    print("- Accessing local variable:", local_var)

scope_demo()

print("\nOutside function:")
print("- Accessing global variable:", global_var)

# Uncommenting the line below will raise a NameError because local_var does not exist globally!
# print(local_var)
```
```
Inside function:
- Accessing global variable: I am global
- Accessing local variable: I am local

Outside function:
- Accessing global variable: I am global
```
### 5.2 Modifying Global Variables (The `global` Keyword)
By default, a function can read global variables, but trying to modify them will create a new local variable of the same name. To modify a global variable inside a function, you must declare it using the `global` keyword.
### 5.3 Nested Scope (The `nonlocal` Keyword)
When you have nested functions, the inner function can access variables in the outer function's scope. If you want to modify a variable in the outer function from the inner function, use the `nonlocal` keyword.
