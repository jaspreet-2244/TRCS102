# Data Science:
## What is Data Science?
Data Science is a dynamic and rapidly growing field. At its core, **Data Science is the process of collecting, cleaning, analyzing, and interpreting data to solve real-world problems and support decision-making.**
### 👥 Discussion Starter: Think About It!
Take a moment to consider how technology impacts your daily life:
* **How does Netflix recommend movies?** It analyzes your watch history and compares it with similar users.
* **How does Amazon recommend products?** It looks at your browsing patterns, purchase history, and items frequently bought together.
* **How does Google Maps predict traffic?** It aggregates real-time location data from millions of smartphones to calculate speeds and detect congestion.

All of these are powered by **Data Science**!

---

## Real-World Applications
Data Science is used in almost every industry today:

1. 🏥 **Healthcare**
   * **Disease Prediction:** Identifying risks of diseases before they become severe.
   * **Patient Analysis:** Tracking patient recovery rates and customizing treatments.

2. 🏦 **Banking & Finance**
   * **Fraud Detection:** Identifying unusual transaction patterns in real-time.
   * **Loan Approval:** Assessing creditworthiness based on customer history.

3. 🛒 **E-Commerce**
   * **Product Recommendations:** Suggesting items you might like.
   * **Customer Behavior Analysis:** Understanding why customers buy certain products.

4. ⚽ **Sports**
   * **Player Performance:** Tracking speed, stamina, and gameplay statistics.
   * **Match Statistics:** Helping coaches build winning strategies.

5. 📱 **Social Media**
   * **Content Recommendations:** Customizing your feed (Instagram, TikTok, YouTube).
   * **Ad Targeting:** Showing you advertisements that match your interests.

---

## Why Data Science?
Organizations generate massive amounts of data every single day. Without Data Science, this data is just noise.

Data Science helps organizations:
* **Make better, data-driven decisions** (rather than guessing).
* **Improve customer experience** by personalizing services.
* **Increase efficiency** by identifying bottlenecks.
* **Reduce costs** by optimizing operations.
* **Identify hidden trends and patterns** in market data.
* **Increase revenue** through smarter pricing and products.
#  Data Science Lifecycle:

Every successful Data Science project follows a structured, step-by-step process. This is known as the **Data Science Lifecycle**.

```text
    Business Problem
           │
           ▼
    Data Collection
           │
           ▼
     Data Cleaning
           │
           ▼
     Data Analysis
           │
           ▼
    Model Building
           │
           ▼
      Evaluation
           │
           ▼
       Deployment
           │
           ▼
       Monitoring
```

Let's break down each step using a real-world example: **A Telecom company wants to reduce customer churn (customers leaving the service).**

---

### Step 1: Business Problem 🎯
First, we must define the problem and ask the right questions.
* **Example:** The telecom company wants to identify customers likely to leave.
* **Core Question:** *"Which customers are likely to churn (cancel their subscription)?"*

### Step 2: Data Collection 📥
Next, we gather relevant data from various sources.
* **Data Sources:**
  * Databases (SQL databases)
  * Excel files or CSV spreadsheets
  * APIs (web services)
  * Websites (web scraping)
  * Mobile applications (app logs)

### Step 3: Data Cleaning 🧹
Real-world data is messy, incomplete, and contains errors. We must clean it.
* **Common Problems:**
  * **Missing Values:** Age is unknown (`?` or `NaN`), or Salary is `NULL`.
  * **Duplicate Records:** The same customer is listed twice in the database.
  * **Incorrect Entries:** Age is listed as `-5` or a negative salary.

### Step 4: Data Analysis (EDA) 📊
We explore the clean data to find initial trends, summaries, and insights.
* **Questions we might ask:**
  * What is the average age of our customers?
  * Which data plans sell the most?
  * Which city generates the highest revenue?
  * Do customers with higher bills leave more often?

### Step 5: Model Building 🤖
We use historical data to build mathematical or machine learning models that can make predictions.
* **Example:** Build a predictive model that takes customer usage details and predicts whether they will churn (Yes or No).

### Step 6: Evaluation 📈
We test the model on new data to see how accurate it is before deployment.
* **Example:** Checking if the model is correct. *"Our churn model is 92% accurate!"*

### Step 7: Deployment 🚀
We integrate the model into a live application where business users can use it.
* **Example:** Deploying the model to the customer service system so agents see an alert when a churning customer calls.

### Step 8: Monitoring 🔍
We continuously monitor the model's performance over time. Because customer behavior and business environments change, the model must be periodically updated.

# Module 4: Python Fundamentals - Variables

## What is a Variable?
A **variable** is a container used to store data in a computer's memory. Think of a variable as a **labeled storage box**. You put data inside the box and give the box a name so you can find it later.

Here is how we declare variables in Python:
```python
name = "Rahul"        # Stores text (String)
age = 25              # Stores a whole number (Integer)
salary = 50000        # Stores an integer or decimal number
```

---

## Variable Naming Rules ⚠️
To make sure Python understands your variables, follow these naming rules:

### ✅ Valid Names:
* `student_name = "Rahul"` (Can use underscores)
* `age1 = 25` (Can contain numbers, but not start with them)
* `salary_amount = 50000` (Descriptive names)

### ❌ Invalid Names:
* `1age = 25` (Cannot start with a number)
* `student-name = "Rahul"` (Cannot use hyphens `-`)
* `student name = "Rahul"` (Cannot contain spaces)

### 💡 Best Practices:
* **Use meaningful and descriptive names:** Write `employee_salary` instead of just `s`.
* **Use lowercase with underscores (snake_case):** This is the standard style for Python variables.
# Module 5: Input and Output

## 1. Output Using `print()`
The `print()` function displays text or calculations on the screen. Let's look at different ways to print.

### A. Basic Print
Prints simple values or variables.
```python
print("Hello World")
```

### B. Printing Multiple Values
You can print multiple variables separated by commas. Python automatically adds a space between them.
```python
name = "Rahul"
age = 25
print(name, age)
```

### C. The `sep` Parameter (Separator)
By default, multiple values are separated by spaces. You can change this using `sep`.
```python
print("Python", "Data", "Science", sep="-")
```

### D. The `end` Parameter
By default, `print()` ends with a new line. You can change what it prints at the end (like a space or comma) using `end`.
```python
print("Hello", end=" ")
print("World")
```

### E. Escape Characters
* `\n`: Moves to a **New Line**.
* `\t`: Adds a **Tab Space**.
```python
print("Hello\nWorld")
print("Name\tAge")
```

### F. String Concatenation
Joining strings together using the `+` operator. (Note: both values must be strings!).
```python
name = "Rahul"
print("My name is " + name)
```

### G. f-Strings (Formatted Strings) 🌟
**Highly Recommended!** f-strings are the easiest and cleanest way to print variables inside text. Simply put an `f` before the quotes, and place variables inside curly braces `{}`.
```python
name = "Rahul"
age = 25
print(f"My name is {name} and I am {age} years old.")
```

# Module 6: Python Data Types

## Overview
A **data type** defines the kind of value stored in a variable. In Data Science, understanding data types helps us process clean datasets.

We will cover six core data types:
1. **Integer (`int`):** Whole numbers (e.g., 25, -100).
2. **Float (`float`):** Decimal numbers (e.g., 45000.75, 3.14).
3. **String (`str`):** Text data (e.g., "Rahul").
4. **List (`list`):** Ordered collections of items (e.g., `["Apple", "Mango"]`).
5. **Tuple (`tuple`):** Ordered, fixed collections of items (e.g., `(1, 2, 3)`).
6. **Set (`set`):** Unordered collections of unique items (e.g., `{"Apple", "Mango"}`).

---

## Mutable vs Immutable 🔄
* **Immutable (Cannot change):** Once created, their values cannot be altered. If you modify them, Python creates a new object in memory.
  * *Examples:* `int`, `float`, `str`, `tuple`
* **Mutable (Can change):** You can add, remove, or modify items after creation without creating a new object in memory.
  * *Examples:* `list`, `dict`, `set`

### Mutability Summary Table
| Data Type | Keyword | Mutable / Immutable |
| :--- | :--- | :--- |
| **Integer** | `int` | **Immutable** |
| **Float** | `float` | **Immutable** |
| **String** | `str` | **Immutable** |
| **List** | `list` | **Mutable** |
| **Tuple** | `tuple` | **Immutable** |
| **Set** | `set` | **Mutable** |
| **Dictionary** | `dict` | **Mutable** |

---

* **Float (`float`):** Stores decimal numbers. Immutable.

## 2. String (`str`)
Strings store text. They are written inside single quotes `'...'` or double quotes `"..."`.

### Indexing & Slicing 
Since strings are ordered sequences of characters, we can extract specific parts:
* **Indexing:** Accessing a single character by its position (starts at `0` for the first character, and `-1` for the last character).
* **Slicing:** Accessing a range of characters using `[start:end]` (where the `end` position is excluded).

### Common String Functions:
* `len(text)`: Returns the length (number of characters).
* `text.upper()`: Converts text to UPPERCASE.
* `text.lower()`: Converts text to lowercase.
* `text.replace(old, new)`: Replaces a word/character with another.
* `text.startswith(prefix)`: Checks if the string starts with a specific prefix (returns `True` or `False`).
