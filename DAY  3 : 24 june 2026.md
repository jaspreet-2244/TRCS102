## 5. Sets (`set`)
A **set** is a mutable, unordered collection of unique elements.

### Characteristics:
- **Mutable**: Elements can be added or removed, though set elements themselves must be hashable (immutable).
- **Unordered & Unindexed**: Elements have no defined order, and elements cannot be retrieved by index or sliced.
- **Unique Elements**: Duplicate items are automatically removed upon creation or insertion.
- **Mathematical Operations**: Highly optimized for set math like unions, intersections, and differences.

### Common Set Methods:

| Method | Description | Operator Equivalent |
| :--- | :--- | :--- |
| `.add(x)` | Adds element `x` to the set | N/A |
| `.update(iterable)` | Adds all elements from iterable to the set | N/A |
| `.remove(x)` | Removes element `x` (raises `KeyError` if not found) | N/A |
| `.discard(x)` | Removes element `x` if present (does NOT raise error if missing) | N/A |
| `.pop()` | Removes and returns an arbitrary element | N/A |
| `.union(other)` | Returns the union of two sets | `set1 \| set2` |
| `.intersection(other)` | Returns the intersection of two sets | `set1 & set2` |
| `.difference(other)` | Returns elements in the calling set but not in `other` | `set1 - set2` |
| `.symmetric_difference(other)` | Returns elements in either set, but not both | `set1 ^ set2` |
| `.issubset(other)` | Checks if all elements of this set are in `other` | `set1 <= set2` |
| `.issuperset(other)` | Checks if this set contains all elements of `other` | `set1 >= set2` |
| `.isdisjoint(other)` | Checks if two sets have no common elements | N/A |
| `.clear()` | Removes all elements from the set | N/A |

### Built-in Functions with Sets:
- `len(s)`: Returns the cardinality (number of elements) of the set.
- `set(iterable)`: Converts an iterable to a set (often used for deduplication).
- `max(s)` / `min(s)` / `sum(s)`: Standard numerical operations on elements.
- `sorted(s)`: Returns a new *sorted list* containing the set's elements.

# Operators in Python
 an **operator** is a special symbol that directs the computer to perform specific mathematical, comparison, or logical manipulations on operands (variables and values). For example, in the expression `x + y`, the `+` is the operator and `x` and `y` are the operands.
### Learning Objectives:
In this notebook, we will cover:
1. **Arithmetic Operators**: Performing basic math.
2. **Comparison (Relational) Operators**: Comparing values.
3. **Logical Operators**: Combining conditional statements.
4. **Assignment Operators**: Storing and modifying variables.
5. **Identity Operators (`is`, `is not`)**: Comparing memory locations (objects).
6. **Membership Operators (`in`, `not in`)**: Finding items in sequences and mappings.

> ** Excluded Topics:** Bitwise (binary) and shift operators (`&`, `|`, `^`, `~`, `<<`, `>>`) are specialized tools for low-level memory operations and are skipped in this general-purpose tutorial.

##  Arithmetic Operators
Arithmetic operators are used to perform mathematical operations such as addition, subtraction, multiplication, and division.
Here are the 7 arithmetic operators available in Python:
| Operator | Name | Example | Explanation |
| :--- | :--- | :--- | :--- |
| `+` | Addition | `a + b` | Adds two operands |
| `-` | Subtraction | `a - b` | Subtracts the right operand from the left |
| `*` | Multiplication | `a * b` | Multiplies two operands |
| `/` | Division | `a / b` | Divides left operand by right operand (always returns a float) |
| `%` | Modulus | `a % b` | Returns the division remainder |
| `**` | Exponentiation | `a ** b` | Left operand raised to the power of the right (`a` to the power `b`) |
| `//` | Floor Division | `a // b` | Divides and rounds down to the nearest whole number (returns an integer if inputs are integers) |

##  Comparison (Relational) Operators
Comparison operators are used to compare two values. They return a Boolean value: either `True` or `False` depending on whether the relationship holds.
| Operator | Name | Example | Explanation |
| :--- | :--- | :--- | :--- |
| `==` | Equal | `x == y` | `True` if operands are equal |
| `!=` | Not Equal | `x != y` | `True` if operands are not equal |
| `>` | Greater Than | `x > y` | `True` if left operand is greater than right |
| `<` | Less Than | `x < y` | `True` if left operand is less than right |
| `>=` | Greater Than or Equal To | `x >= y` | `True` if left operand is greater than or equal to right |
| `<=` | Less Than or Equal To | `x <= y` | `True` if left operand is less than or equal to right |
---
##  Logical Operators
Logical operators are used to combine conditional statements. Python has three logical operators: `and`, `or`, and `not`.

### 1. The `and` Operator
Evaluates to `True` **only if both** operands are `True`.
- `True and True` $\rightarrow$ `True`
- `True and False` $\rightarrow$ `False`
- `False and False` $\rightarrow$ `False`

### 2. The `or` Operator
Evaluates to `True` **if at least one** of the operands is `True`.
- `True or False` $\rightarrow$ `True`
- `False or False` $\rightarrow$ `False`

### 3. The `not` Operator
Reverses the Boolean state of its operand.
- `not True` $\rightarrow$ `False`
- `not False` $\rightarrow$ `True`

## Identity Operators
Identity operators compare the memory locations (or references) of two objects. In Python, variables are labels that point to objects in memory. 

The two identity operators are:
1. `is`: Returns `True` if both variables point to the exact same object in memory.
2. `is not`: Returns `True` if both variables point to different objects in memory.

###  The Key Difference: `==` vs. `is`
- **`==` (Equality)**: Compares the **values** of the objects (Do they contain the same data?).
- **`is` (Identity)**: Compares the **memory addresses** of the objects (Are they literally the same object in memory?).

To check where an object is stored in memory, Python provides the built-in `id()` function, which returns a unique integer identifier for the object.
---
## Membership Operators
Membership operators are used to test whether a value or variable is found inside a sequence (such as a string, list, tuple, set, or dictionary).

The two membership operators are:
1. `in`: Returns `True` if the specified value is present in the sequence.
2. `not in`: Returns `True` if the specified value is **not** present in the sequence.

### Checking Membership in Different Collections:
- **Strings**: Checks for sub-strings (e.g., `'a'` in `'apple'`).
- **Lists/Tuples/Sets**: Checks for element presence.
- **Dictionaries**: Checks if the item exists **as a key** in the dictionary (not as a value).

> ** Performance Note:** Finding membership in a **List** or **Tuple** requires scanning the collection element-by-element ($O(N)$ linear time). Checking membership in a **Set** or **Dictionary** uses hash tables and is instantaneous ($O(1)$ constant time), regardless of size!
